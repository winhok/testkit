# 测试用例列定义（Excel / CSV）

与 apitestspec-scenario-runner 共享的列约定，导出与兼容导入时必须一致。列定义同时适用于 **Excel 与 CSV**；composer 默认可导出 Excel，runner 支持两种格式作为兼容输入。

## 基础列（按顺序）

| 列名 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 用例ID | 文本 | 是 | 唯一标识，如 TC_001 |
| 用例名称 | 文本 | 是 | 简短描述 |
| 接口路径 | 文本 | 是 | 如 /api/v1/user，占位符用 {参数名} 如 /api/v1/users/{id} |
| 路径参数 | 文本 | 否 | JSON 对象，键为占位符名、值为字面量；runner 另支持占位符 __RANDOM__、__RANDOM_INT__、__DEP:依赖名:字段名__（生成时请勿使用，一律写死参数）。如 {"id":"1"} |
| 请求方法 | 文本 | 是 | GET / POST / PUT / DELETE |
| 请求体/参数 | 文本 | 否 | JSON 或 query 参数；runner 另支持请求体内 __RANDOM__（生成时请勿使用，一律写死参数）。需唯一值时用固定示例如 user001、test@demo.com |
| 预期状态码 | 数字 | 是 | 如 200 |
| 预期响应校验 | 文本 | 否 | JSONPath 或简单断言规则 |
| 优先级 | 文本 | 否 | P0 / P1 / P2 |

## 依赖列（前置依赖支持）

| 列名 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| 前置依赖 | 文本 | 是 | 依赖标识，无依赖填「无」 | user_login / 无 |
| 依赖产出提取 | 文本 | 条件 | 有前置依赖时填写 | $.data.token |
| 注入方式 | 文本 | 条件 | 有前置依赖时填写 | Header:Authorization:Bearer {token} / Cookie |

## 执行控制列

| 列名 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 是否运行 | 文本 | 否 | 是/1/Y/true 表示执行，否/0/N/false 表示跳过；空视为「是」 |
| 运行结果 | 文本 | 否 | 执行前可为空；执行结束后由 runner 写回 PASS 或 FAIL |

## 示例行

| 用例ID | 用例名称 | 接口路径 | 路径参数 | 请求方法 | 请求体/参数 | 预期状态码 | 预期响应校验 | 优先级 | 前置依赖 | 依赖产出提取 | 注入方式 | 是否运行 | 运行结果 |
|--------|----------|----------|----------|----------|-------------|------------|--------------|--------|----------|-------------|----------|----------|----------|
| TC_001 | 登录成功 | /auth/login | | POST | {"username":"user","password":"123"} | 200 | $.code==200 | P0 | 无 | | | 是 | |
| TC_002 | 获取用户信息 | /api/v1/user | | GET | | 200 | $.data.userId | P0 | user_login | $.data.token | Header:Authorization:Bearer {token} | 是 | |
| TC_003 | 获取指定用户 | /api/v1/users/{id} | {"id":"1"} | GET | | 200 | $.code==200 | P0 | user_login | $.data.token | Header:Authorization:Bearer {token} | 是 | |

## JSON 输入格式

生成脚本的输入为 **JSON 文件**，支持两种根结构：

- **数组**：`[ { "用例ID": "TC_001", ... }, ... ]`
- **对象带 cases**：`{ "cases": [ { "用例ID": "TC_001", ... }, ... ] }`

每项对象的键与上表列名一致（用例ID、用例名称、接口路径、路径参数、请求方法、请求体/参数、预期状态码、预期响应校验、优先级、前置依赖、依赖产出提取、注入方式、是否运行、运行结果）。缺失键在 CSV 中填空；`预期状态码` 可为数字或数字字符串。路径参数为空时可不填或填空字符串。

示例：

```json
[
  {"用例ID": "TC_001", "用例名称": "登录成功", "接口路径": "/auth/login", "请求方法": "POST", "请求体/参数": "{\"username\":\"admin\",\"password\":\"admin123\"}", "预期状态码": 200, "预期响应校验": "$.code==200", "优先级": "P0", "前置依赖": "无", "依赖产出提取": "", "注入方式": "", "是否运行": "是", "运行结果": ""},
  {"用例ID": "TC_002", "用例名称": "登录缺少用户名", "接口路径": "/auth/login", "请求方法": "POST", "请求体/参数": "{\"password\":\"x\"}", "预期状态码": 400, "预期响应校验": "$.code==400", "优先级": "P1", "前置依赖": "无", "依赖产出提取": "", "注入方式": "", "是否运行": "是", "运行结果": ""}
]
```

## 用例设计（正常与异常）

正常与异常用例均由大模型根据接口文档设计，并输出为上述 JSON，由脚本统一转为 CSV。脚本不解析文档、不按接口类型追加任何用例。

## 注意事项

- CSV 编码为 UTF-8；若需 Excel 双击打开无乱码，生成脚本可选用 UTF-8-BOM。
- 第一行为表头，数据从第二行开始；含逗号、双引号、换行的字段按 RFC 4180 用双引号包裹。
- 空字段表示该列无值。
- 依赖配置（URL、账号密码）在 config/dependencies.yaml，不写在 CSV 中。
- **生成用例时**路径参数与请求体均使用字面量，不生成 `__RANDOM__`、`__RANDOM_INT__`、`__DEP:...` 占位符（上述占位符由 runner 支持，仅供高级用户手动编辑 CSV 时使用）。需唯一值的字段使用固定示例值（如 user001、test@demo.com），不使用占位符。
